Memory Forensics Analysis: Amadey Trojan Investigation

Endpoint Memory Forensics Project

Comprehensive forensic analysis of a Windows 7 memory dump infected with Amadey Trojan using Volatility3. This investigation reveals malicious processes, command and control communications, and persistence mechanisms.

View Full Project on GitHub

Task 1: Process Enumeration & Malicious Process Identification

Analysis Steps:

Used Volatility3's windows.pslist plugin to enumerate running processes and identify suspicious activity.

vol.py -f Windows7-Snapshot4.vmem windows.pslist.PsList

Key Findings:

Identified suspicious process lssass.exe (PID 2748) mimicking the legitimate Windows process lsass.exe. The malware uses this deceptive naming to evade detection while performing credential theft activities.

Task 2: Malware Location Analysis

Analysis Steps:

Executed windows.cmdline plugin to determine the exact file system location of the malicious process.

vol.py -f Windows7-Snapshot4.vmem windows.cmdline --pid 2748

Key Findings:

Located malware at: C:\Users\0XSH3R~1\AppData\Local\Temp\925e7e99c5\lssass.exe. The use of the Temp folder and short filename format indicates deliberate obfuscation techniques commonly used by malware.

Task 3: Network Communication Analysis

Analysis Steps:

Used windows.netscan to identify network connections and potential C2 communications.

vol.py -f Windows7-Snapshot4.vmem windows.netscan | grep 2748

Key Findings:

Discovered active connections to C2 server: 41.75.84.12:80. The use of port 80 (HTTP) suggests the malware is disguising its communications as normal web traffic to evade detection.

Task 4: Memory Dump & String Analysis

Analysis Steps:

Dumped process memory using windows.memmap and analyzed strings for HTTP requests.

vol.py -f Windows7-Snapshot4.vmem windows.memmap --pid 2748 --dump
strings pid.2748.dmp | grep "GET /"

Key Findings:

Identified HTTP GET requests for additional payloads including DLL files, confirming the malware's modular nature and ongoing communication with C2 servers for payload delivery.

Task 5: File System Artifact Investigation

Analysis Steps:

Scanned for malicious files using windows.filescan with targeted grep patterns.

vol.py -f Windows7-Snapshot4.vmem windows.filescan | grep -E "cred64|clip64"

Key Findings:

Located downloaded payloads: cred64.dll and clip64.dll in AppData Roaming directory. These modular components represent the malware's credential theft and clipboard monitoring capabilities.

Task 6: Persistence Mechanism Investigation

Analysis Steps:

Investigated additional persistence mechanisms using comprehensive file scanning.

vol.py -f Windows7-Snapshot4.vmem windows.filescan | grep -E "lssass"

Key Findings:

Discovered malware artifacts in multiple system locations including Tasks and Temp directories, indicating sophisticated persistence mechanisms designed to maintain access despite system reboots or cleanup attempts.

Investigation Conclusion

The memory forensics analysis successfully uncovered the complete Amadey Trojan infection chain, including initial compromise vector, C2 communications, payload delivery mechanisms, and persistence techniques. The investigation demonstrates the importance of comprehensive memory analysis in modern incident response.
